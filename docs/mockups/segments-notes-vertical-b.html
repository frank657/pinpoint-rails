Pinpoint · segment ↔ note timeline — locked model (ADR 0011)

All scenarios in one view

Notes auto-map into the closed segment that contains their start; open-ended segments and gaps map nothing. A note in a card is assigned (pinned); loose = a timed orphan with no segment; unanchored = no timestamp. Edit mode reveals drag handles + a detach (×) button. Shown below with edit mode on. #4 unanchored untimed (rich-text) notes pin to a group — never on the timeline.

#1 in segment auto-mapped by start into the closed segment that contains it.

#3 loose a timed note in a gap / before any segment → dashed ghost row, in time order.

span a range note covering several segments still lives in one card (its start segment); the reach is shown with a cosmetic “↳ range reaches…”. No wrapping bracket — matches the stored model.

straddle a range crossing a boundary lives in its start segment with “↳ extends past…”. Not duplicated.

#16 moved a note dragged into a segment its time is outside shows a “⇄ moved here” chip; it sorts by its timestamp.

#8 empty a closed segment with no notes still shows its card.

#5 #13 open-ended header reads “7:00 →”. It auto-maps nothing; the 7:30 note stays loose. Notes only enter by drag (the “drilling cue”), or once an end is set.

edit mode ⠿ handles + × detach appear only when editing; toggling off leaves a clean read/seek list.

Faithful to ADR 0011: a note is in a segment iff seg.start ≤ note.start < seg.end on a closed segment (earliest-created wins on overlap); open-ended/gap/untimed map nothing; placement is sticky (editing times never reshuffles); drag re-pins, × detaches (→ loose). Span & straddle are cosmetic only — every note is stored under exactly one segment (or none).
